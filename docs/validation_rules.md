# 검증 규칙

Synapse Engine은 아래 규칙으로 패치 후보를 판정합니다.

1. **DAG/인과 규칙**
   - self-loop 금지
   - 사이클이 있으면 실패(F001)
   - 고립 노드 점검

2. **보존 규칙**
   - 이벤트 에너지/엔트로피/정보 합계 구간과 관측 bound 비교
   - 교집합이 비어 있으면 실패(F003)

3. **엔트로피 규칙**
   - 위상 순서를 따라 누적 엔트로피가 감소하면 실패(F004)

4. **Lightcone 규칙**
   - 유효하지 않은 cone 조합이나 존재하지 않는 노드 참조 시 실패(F005)

5. **민감도 규칙**
   - 관측 불확실성으로 계산한 민감도 값이 임계치 초과 시 실패(F007)

6. **정책 규칙**
   - `external_output_zero`에서 외부 출력 관계가 존재하면 실패(F012)

7. **타입 규칙**
   - 이벤트 타입 미지원 시 실패(F009)
